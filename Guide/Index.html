01

Clone the Repository

Start by cloning the project and navigating into it. You'll configure environment variables in the next step.

bash 
git clone https://github.com/Okashanadeem/Quizzy
cd Quizzy
02

Configure Your Environment

Create a .env file inside the backend/ directory and fill in all required values.

backend/.env 
PORT=5000
MONGODB_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/Quizzy?retryWrites=true&w=majority
JWT_SECRET=your_jwt_secret_key
GMAIL_USER=your_email@gmail.com
GMAIL_APP_PASSWORD=your_16_digit_app_password
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123
CLIENT_URL=http://localhost:3000  # update to Vercel URL after deployment
⚠ Before deploying

Update CLIENT_URL and MONGODB_URI with your live Vercel and Atlas URLs. Do not push your .env file to GitHub.

🍃 Getting your MongoDB Atlas URI

1

Create a free cluster

Log in to MongoDB Atlas and create a free Shared Cluster.

2

Add a database user

Go to Database Access → create a user with Read/Write permissions. Avoid special characters (@ # $) in the password, or URL-encode them.

3

Allow network access

Go to Network Access → Add IP Address → choose Allow Access from Anywhere (0.0.0.0/0).

4

Copy the connection string

Go to Deployment → Database → Connect → Drivers. Copy the URI and add /Quizzy before the ? to name your database.

03

Set Up Gmail App Password

Quizzy emails result sheets to the instructor automatically. You must use a Gmail App Password — not your regular account password.

1

Enable 2-Step Verification

Visit Google Account Security and turn on 2-Step Verification.

2

Find App Passwords

Search for App Passwords in your Google Account's search bar and open the setting.

3

Create a new app password

Select Other, name it Quizzy, and click Generate.

4

Paste into .env

Copy the 16-digit code and set it as your GMAIL_APP_PASSWORD in backend/.env.

✓ Tip

The App Password is only shown once. Copy it immediately. If you lose it, you'll need to generate a new one.

04

Manage the Student List

Students log in using credentials stored in a JSON file. Only students listed here can access the exam portal.

Edit this file: backend/src/data/students.json

[
  {  "id": "BSE-25F-001",  "name": "Okasha Nadeem"  },

⚠ Important

After editing this file, you must commit and push the changes to GitHub so the deployed frontend picks up the new students.

05

Deploy to Vercel

Quizzy's backend and frontend are deployed as two separate Vercel projects. Follow the order below.

STEP 1

Push to GitHub

Upload your configured project to a GitHub repository (public or private).

git push origin main
STEP 2

Deploy Backend

On Vercel, import the backend/ folder. Add all .env variables in the Environment Variables section. Copy the deployment URL.

STEP 3

Deploy Frontend

Import the frontendApp/ folder. Add one environment variable: NEXT_PUBLIC_API_URL = your backend URL from Step 2.

ℹ After deploying

Go back to your backend/.env and update CLIENT_URL with your live frontend URL. Redeploy the backend for CORS to work correctly.

Frontend environment variable 
NEXT_PUBLIC_API_URL=https://quizzy-api.vercel.app
